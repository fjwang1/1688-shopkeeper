## 核心能力

本技能掌握以下专业能力：

### 1.1 商品搜索（选品）
- **自然语言理解**：从用户描述中提取关键词、价格区间、品类偏好
- **多维度筛选**：支持按渠道、类目、数量筛选
- **结果格式化**：将API返回转换为易读的商品列表

### 1.2 店铺管理
- **绑定店铺查询**：获取用户已绑定的下游店铺列表
- **授权状态诊断**：识别店铺授权是否过期，给出处理建议
- **平台特性了解**：熟悉各平台的入驻要求和运营特点

### 1.3 铺货执行
- **批量铺货**：支持单次最多50个商品的批量铺货
- **渠道适配**：自动匹配商品到对应平台格式
- **结果追踪**：区分成功/失败，提供重试建议

### 1.4 用户引导
- **新手引导**：AK获取 → 开店绑定 → 首次选品 → 首次铺货
- **问题诊断**：根据错误信息判断原因并给出解决方案
- **最佳实践**：基于场景提供选品和铺货建议

## 二、知识体系

### 2.1 认证机制（AK）

AK（Access Key）是访问 1688 平台的身份凭证。

#### 获取 AK

1. 下载 1688 AI版 APP
2. 登录后进入「我的」→「开发者设置」
3. 复制 AK（一串字母数字组合，通常40+位）

#### 配置 AK

本 skill 通过 **OpenClaw SecretRef** 读取 AK。配置一次，所有会话生效。

**方式一：对话中配置（推荐）**

直接在飞书/对话中告诉我：
```
我的AK是 xxx...xxx
```

我会立即：
1. ✅ 保存 AK 到配置文件（持久化）
2. ✅ 设置当前会话环境变量（立即生效）
3. 🔄 **尝试继续你的选品/铺货请求**

如果当前会话能正常使用，你会看到结果；
如果遇到权限错误，我会提示你重启 Gateway。

**重启后**，所有新会话都能自动使用此 AK，无需再次配置。

**方式二：手动配置 openclaw.json**

编辑 `~/.openclaw/openclaw.json`：

```json5
{
  // ... 其他配置
  env: {
    ALI_1688_AK: "your_ak_here"
  }
}
```

然后重启 Gateway。

**方式三：使用配置脚本**

```bash
cd ~/.openclaw/skills/1688-od-search/scripts
python configure.py your_ak_here
```

**方式四：Gateway 启动参数**

```bash
# ~/.zshrc 或 ~/.bashrc
export ALI_1688_AK=your_ak_here

# 使配置生效
source ~/.zshrc
```

⚠️ **注意**：方式三只对从当前 shell 启动的 Gateway 有效。如果 Gateway 作为系统服务运行（LaunchAgent/systemd），需要使用方式一或二。

#### 安全提示

- AK 仅用于本地 API 调用，不会上传到任何服务器
- 建议定期更换 AK（可在 1688 AI版 APP 中重新生成）
- 不要将 AK 提交到代码仓库

### 2.2 下游渠道特性

| 渠道 | 特点 | 入驻门槛 | 适合商品 | 注意事项 |
|------|------|---------|---------|---------|
| **抖音** | 内容电商，流量大 | 需营业执照 | 新奇特、视觉冲击力强的商品 | 重视视频素材 |
| **拼多多** | 低价走量 | 个人可入驻 | 性价比高的日用百货 | 价格敏感 |
| **小红书** | 种草社区 | 需品牌资质 | 美妆、生活方式 | 重视图文质量 |
| **淘宝** | 传统电商 | 需保证金 | 全品类 | 竞争激烈 |

### 2.3 选品策略

#### 好卖商品的特征
1. **应季性**：当前季节需要的商品（如夏天卖风扇）
2. **差异化**：与竞品有明显区别（功能/外观/价格）
3. **供应链稳定**：1688店铺评分高，发货快
4. **利润空间**：采购价与售价之间有足够毛利

#### 避坑指南
- ❌ 避免侵权商品（品牌仿品）
- ❌ 避免过重/过大商品（运费成本高）
- ❌ 避免保质期短的食品
- ❌ 避免售后率高的电子产品

### 2.4 常见错误及处理

| 错误现象 | 可能原因 | 解决方案 |
|---------|---------|---------|
| "AK未配置" | 环境变量未设置 | 提示用户配置 ALI_1688_AK |
| "店铺授权过期" | 长时间未使用或密码修改 | 引导用户在1688 AI版APP重新授权 |
| "选品返回为空" | 关键词太冷门或无货源 | 建议更换关键词或扩大搜索范围 |
| "铺货失败" | 商品信息不完整或平台审核 | 检查商品详情，等待审核或换品 |
| "网络超时" | API响应慢或网络问题 | 自动重试，最多3次 |

   - 有可用店铺？

## 四、API 使用参考

### 4.1 核心函数

```python
from scripts.api import search_products, list_bound_shops, publish_items

# 选品
products = search_products(
    query="帮我在1688找支持一件代发包邮的连衣裙，100元以内的",      # 搜索关键词
    channel="douyin",       # 下游渠道
    max_results=10          # 返回数量（1-50）
)
# 返回: List[Product]
# Product属性: id, title, price, image, url, sales

# 查店铺
shops = list_bound_shops()
# 返回: List[Shop]
# Shop属性: code, name, channel, is_authorized

# 铺货
result = publish_items(
    item_ids=["123", "456"],    # 商品ID列表
    shop_code="260391138"       # 店铺代码
)
# 返回: PublishResult
# PublishResult属性: success, published_count, failed_items
```

### 4.2 返回值处理

**选品结果处理**：
```python
if products:
    # 格式化展示
    for i, p in enumerate(products[:10], 1):
        print(f"{i}. {p.title} - ¥{p.price}")
    
    # 保存data_id供后续使用
    data_id = save_to_memory(products)
else:
    # 无结果处理
    suggest_alternatives(query)
```

**店铺查询处理**：
```python
valid_shops = [s for s in shops if s.is_authorized]
expired_shops = [s for s in shops if not s.is_authorized]

if expired_shops:
    warn_about_expiration(expired_shops)

if not valid_shops:
    guide_to_bind_shop()
```

**铺货结果处理**：
```python
if result.success:
    celebrate_and_guide_next(result.published_count)
else:
    if result.failed_items:
        analyze_failures(result.failed_items)
        suggest_retry_or_skip()
```

## 五、最佳实践

### 5.1 对话节奏控制

**不要一次性给太多信息**：
- ❌ "这是10个商品，你可以铺货到抖音店或拼多多店，也可以继续搜索其他商品..."
- ✅ "找到10个商品，先看下这些是否符合需求？"

**每步确认后再继续**：
1. 选品完成 → 问"满意吗？"
2. 确认满意 → 问"铺货到哪个店铺？"
3. 确认店铺 → 执行铺货 → 展示结果

### 5.2 错误恢复策略

**网络错误**：
- 自动重试3次（已实现）
- 仍失败 → 提示"网络不稳定，请稍后再试"

**认证错误**：
- AK无效 → 引导重新配置
- 店铺过期 → 引导重新授权

**业务错误**：
- 选品无结果 → 建议换关键词
- 铺货失败 → 分析具体原因

### 5.3 用户体验优化

**首次使用**：
- 耐心解释每个概念（AK、店铺、铺货）
- 提供详细的步骤指引
- 允许用户随时提问

**熟练用户**：
- 快速执行，减少确认步骤
- 记住偏好（常用渠道、店铺）
- 提供快捷指令

## 六、边界与限制

### 6.1 本技能不做的事

- ❌ 不替代用户在电商平台的后台操作
- ❌ 不保证商品一定能卖出
- ❌ 不处理售后和客服问题
- ❌ 不提供市场趋势预测

### 6.2 需要用户自己完成的

- 在电商平台注册账号
- 处理买家订单和售后
- 决定定价策略
- 遵守平台规则和政策

### 6.3 技术限制

- 单次选品最多50个商品
- 单次铺货最多50个商品
- API调用频率受限于1688平台
- 部分功能需要有效的AK和授权
